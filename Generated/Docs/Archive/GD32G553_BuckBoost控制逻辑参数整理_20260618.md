# GD32G553 Buck-Boost 控制逻辑与参数整理

日期：2026-06-18  
范围：`GD32G553_version_2_1` 固件，结合外层 `hardware` 目录中的功率板/主控板网表、原理图导出文本和 BOM。

这份文档只整理固件真正用到的参数、判断条件和计算公式。电感纹波等推导只作为联调参考，不作为当前闭环控制的实时计算路径。

## 1. 总体结构

当前功率级是四开关非反相 Buck-Boost：

- 输入侧半桥：`PWM1H/PWM1L`，固件里记为 Buck 半桥。
- 输出侧半桥：`PWM2H/PWM2L`，固件里记为 Boost 半桥。
- 两个半桥驱动芯片：功率板 BOM 中 U3/U4 为 `EG3112`。
- 主电感：功率板 BOM 中 L2 为 `22uH`。
- 输入/输出检流电阻：功率板 BOM 中 R11/R12 为 `5mΩ`。

控制链路可以按下面理解：

```text
功率板采样网络
  -> MCU ADC 原始值
  -> adc_manager.c 换算 Vin/Vout/Iin/Iout/温度
  -> charge_manager.c 充电状态机和目标值
  -> power_manager.c 输入功率/温度/降额限流
  -> power_control.c 电流环/电压环 PI
  -> Buck/Boost 区域映射
  -> HRTIMER PWM1/PWM2 输出
```

## 4. ADC 与反馈计算

### 4.1 电压换算

`adc_manager.c` 当前计算：

```c
adc_pin_mv = raw * 3300 / 4095;
terminal_mv = adc_pin_mv * gain_x1000 / 1000;
```

对应当前固件：

```text
Vin  = raw_vin  * 3300 / 4095 * 16000 / 1000
Vout = raw_vout * 3300 / 4095 * 16000 / 1000
```

硬件依据：

- 功率板网表/BOM：电压采样分压使用 `75kΩ` 和 `4.7kΩ`，原理图导出文本标注“电压采样放大倍数=4.7/75=0.062667”。
- 理论反算系数约 `75/4.7=15.96`；如果按完整分压式 `(75+4.7)/4.7`，约 `16.96`。
- 当前固件采用 `16.00`，属于已按台架标定后的工程系数。若后续实测 40V 显示偏差明显，应优先在 `BMS_ADC_VIN_GAIN_X1000/BMS_ADC_VOUT_GAIN_X1000` 做标定，而不是改控制逻辑。

### 4.2 电流换算

`adc_manager.c` 当前计算：

```c
diff_raw = raw - offset_raw;
adc_delta_mv = diff_raw * 3300 / 4095;
current_ma = adc_delta_mv * ma_per_mv_x1000 / 1000;
```

对应当前固件：

```text
Iin  = (raw_iin  - 7) * 3300 / 4095 * 3226 / 1000
Iout = (raw_iout - 6) * 3300 / 4095 * 3226 / 1000
```

硬件依据：

- 输入检流：R11 = `5mΩ`。
- 输出检流：R12 = `5mΩ`。
- 电流运放：U1/U2 = `GS8558-SR`。
- 运放相关电阻：`6.2kΩ/100Ω`，原理图导出文本标注“电流采样放大倍数=6200/100=62”。

所以：

```text
1A -> Rsense 5mV -> 运放输出约 310mV
1mV ADC 电压 -> 1A / 310mV = 3.226mA/mV
```

因此 `3226 mA/V` 的固件系数是匹配硬件的。该系数同时用于输入电流和输出电流。

### 4.3 控制用反馈电流

`power_control.c` 里有两个电流概念：

- `output_current_ma`：始终取功率板 `ADC_Iout` 的正向值，用于软件 OCP。
- `measured_current_ma`：闭环 PI 用的反馈电流。

闭环反馈选择规则：

```text
正常 BMS 充电 + 非预连接 + 非数字电源 + BM2016 电池电流有效
  -> 用 BM2016/R71 的 batteryCurrentMa
否则
  -> 用功率板 ADC_Iout
```

这个设计很重要：真实接电池后，充电电流闭环优先跟电池端电流；预充/预连接或数字电源模式没有可靠电池路径时，才用功率板输出电流。

## 5. 充电目标与状态机

### 5.1 默认参数和限幅

默认充电参数：

```text
targetVoltageMv = 37800
targetCurrentMa = 3000
cutoffCurrentMa = 300
```

参数限幅：

```text
targetCurrentMa: 600mA..3000mA
cutoffCurrentMa: 100mA..300mA
targetVoltageMv: 不超过 37800mV
```

手动数字电源模式单独允许：

```text
targetVoltageMv: 5000mV..40000mV
currentLimitMa: 100mA..3000mA
```

`Power_Control_Clamp_Charge_Target_Mv()` 也做了同样区分：

- 自动充电：目标电压不超过 `37.8V`。
- 数字电源：不按 `37.8V` 限幅，由数字电源参数校验保证不超过 `40V`。

### 5.2 状态切换

自动充电主要状态：

```text
IDLE/PRECHECK
  -> TRICKLE: cellMin <= 3.1V，或启动时 cellMin < 3.2V
  -> CC: cellMin >= 3.2V 且未接近满充
  -> CV: packVoltage + 200mV >= targetVoltage
  -> DONE: CV 且电池路径已闭合、未降额、Vout 接近目标、电池电流 <= cutoff，连续确认
```

当前 CC/CV 的关键点：

- CC 阶段并不固定输出电压，而是电流环控制输出电流。
- CV 阶段把目标电流仍作为最大允许电流传给功率环，真正让电流下降的是电压环。
- `cutoffCurrentMa=300mA` 只用于 CV 完成判断，不作为实时限流目标。

### 5.3 当前目标电流路径

`Charge_Manager_Target_Current_For_State()`：

```text
TRICKLE: min(targetCurrentMa, 200mA)，再经过 Power_Manager_Limit_Current()
CC:      targetCurrentMa，再经过 Power_Manager_Limit_Current()
CV:      targetCurrentMa，再经过 Power_Manager_Limit_Current()
```

`power_control.c` 中还会做软启动斜坡：

```text
s_soft_current_ma 每次最多向目标变化 50mA
预连接阶段不走软斜坡，直接使用目标电流
```

## 6. 输入功率与降额限流

### 6.1 输入功率换算为输出可用电流

`Power_Manager_Calc_Input_Current_Limit()`：

```text
Pin_limit_mW = Vin_mV * inputCurrentLimitMa / 1000
Pout_available_mW = Pin_limit_mW * ETA / 100
若配置 inputPowerLimitW，则 Pout_available_mW 不超过 inputPowerLimitW * 1000
Iout_limit_ma = Pout_available_mW * 1000 / max(Vout, targetVoltage)
```

例子：

```text
Vin = 18V, Iin_limit = 7A, ETA = 92%
Pout_available ~= 18 * 7 * 0.92 = 115.9W
Iout_limit @ 37.8V ~= 3066mA
```

这就是当前 `18V` 最低输入下还能支撑 `37.8V/3A` 的原因。反过来估算满载输入电流：

```text
Iin ~= Vout * Iout / (ETA * Vin)
    ~= 37.8 * 3 / (0.92 * 18)
    ~= 6.85A
```

所以默认输入限流 `7A` 是合理的。

当前 `power_manager.c` 在 CC 状态下有一个实现细节：CC 阶段只把热降额叠到目标电流上，输入电流超限主要交给 `power_control.c` 的 input guard 快速减 duty；非 CC 阶段才把上面的输入功率换算结果作为输出电流上限参与 `Power_Manager_Limit_Current()`。因此联调时看见 CC 阶段 `limitedCurrentMa` 仍接近 3A，并不代表输入保护失效，要同时观察 `input_guard` 后 duty 是否被压低。

### 6.2 输入 foldback

当目标电压高于 `30V` 时，power manager 会监控输入压力：

```text
Vin < 18V
或 Iin > 7A
或转换效率/功率关系异常
  -> 输出允许电流每次下调 100mA，最低 50mA

恢复条件：
Vin >= 19V
且 Iin <= 6.5A
且转换状态不再异常
  -> 每次恢复 25mA，直到退出 foldback
```

### 6.3 温度降额

温度取 MOS 温度和电感温度中较高者：

```text
<= 50C: 不降额
50C..60C: 从请求电流线性降到 minChargeCurrentMa
>= 60C: thermal suspend，目标电流降为 0
```

## 7. 快环控制顺序

`Power_Control_Fast_Loop()` 的主流程：

```text
1. sample 为空、未使能、故障锁定：停止或返回
2. Vin < 18V：输入欠压故障锁存
3. 输出 OVP 检查
4. Vin == 0：停止
5. 输入 guard：快速减小 duty
6. 计算 output_current_ma、measured_current_ma、soft current target
7. 输出 OCP 检查
8. 预连接 coast 处理
9. 判断电压环或电流环
10. PI 计算 step，叠加到 s_power.dutyX100
11. light-load/preconnect/battery-boost/AFE-handover duty 限制
12. Power_Map_Control_To_Pwm() 映射到 Buck/Boost 两路 PWM
13. HRTIMER 应用 PWM
```

电压环判断 `Power_Voltage_Loop_Should_Run()`：

```text
AFE handover: 强制电压环
measured_current_ma > current_ref_ma: 电流环
mode == CV: 电压环
mode == CC/TRICKLE: 电流环
轻载接近目标: 电压环
Vout + 100mV >= targetVoltage: 电压环
否则: 电流环
```

PI 参数：

```text
current kpDiv = 80,  kiDiv = 8000
voltage kpDiv = 100, kiDiv = 10000
单次 step 最大 = 80 x100，占空比单位即 0.80%
```

## 8. Buck/Boost 模式与占空比

### 8.1 区域判断

`Power_Map_Control_To_Pwm()` 使用 `region_target_mv` 判断区域：

```text
if Vin > region_target + 1V:
    Buck 区
elif Vin + 1V < region_target:
    Boost 区
else:
    Buck-Boost 过渡区
```

`region_target_mv` 在 CC/TRICKLE 下不是简单等于 37.8V，而是为了避免空载/轻载过度升压，取：

```text
candidate = max(packVoltage + 800mV, Vout + 800mV)
region_target = min(candidate, targetVoltage)
```

非 CC/TRICKLE 时，`region_target = targetVoltage`。

### 8.2 PWM 映射

控制环输出只有一个 `control_x100`，映射如下：

```text
Buck 区:
    buck_duty  = control_x100
    boost_duty = 0

Boost 区:
    buck_duty  = 95%
    boost_duty = control_x100

Buck-Boost 过渡区:
    buck_duty  = control_x100
    boost_duty = control_x100
```

如果 Vin 被清零但 Vout 还低于目标，固件会强制走 Boost 映射，避免需要升压时误入 Buck。

### 8.3 Boost 前馈公式

固件中 `Power_Control_Boost_Duty_X100()` 用于 light-load、preconnect、AFE handover、battery boost duty 限制：

```text
Dboost = 1 - ETA * Vin / Vo
duty_x100 = 10000 - Vin_mV * ETA_X100 * 100 / target_mV
duty 限幅到 5%..95%
```

例子：

```text
Vin = 18V, Vo = 37.8V, ETA = 92%
Dboost = 1 - 0.92 * 18 / 37.8 = 56.2%
```

### 8.4 Buck 公式的当前角色

理论 Buck 前馈可以写成：

```text
Dbuck = Vo / (ETA * Vin)
```

但当前固件没有把该公式直接塞进 Buck 闭环主路径。Buck 区仍由 PI 输出 `control_x100` 控制输入侧半桥；这样做对现有闭环改动小，实际输出由电流/电压反馈闭环收敛。该公式可用于联调估算初始 duty 或判断异常，不是当前实时控制公式。

## 9. 保护计算

### 9.1 输入欠压/输入 guard

输入欠压：

```text
Vin < 18V -> BMS_FAULT_INPUT_UV，功率级停机/故障锁存
```

输入 guard：

```text
仅 targetVoltage >= 30V 时生效
Vin < 18V 或 Iin > 7A
  -> s_power.dutyX100 -= 120
  -> PI 积分衰减
  -> 立即重新映射 PWM
```

这里的 `120` 是 duty x100 单位，即每次约减小 `1.20%`。

### 9.2 输出过压 OVP

快环 OVP：

```text
自动充电软 OVP: Vout > targetVoltage + 300mV
数字电源软 OVP: Vout > targetVoltage + 2000mV
预连接软 OVP:   Vout > preconnect_ovp_limit，或 targetVoltage + 2000mV

硬 OVP: Vout > min(targetVoltage + 5000mV, 40000mV)
预连接硬 OVP: min(targetVoltage + 5000mV, 40000mV)
```

确认逻辑：

- 硬 OVP 直接把确认计数置满。
- 软 OVP 需要超过软阈值后连续计数，默认 `BMS_OUTPUT_OVP_CONFIRM_SAMPLES=8`。
- 启动时有 `BMS_OUTPUT_OVP_STARTUP_BLANK_SAMPLES=80` 个样本的软 OVP blank；硬 OVP 仍然有效。

`charge_manager.c` 还有一层管理级 OVP：

```text
自动充电: target + 300mV
数字电源: target + 2000mV
硬限制: min(target + 5000mV, 40000mV)
确认计数: BMS_OUTPUT_OVP_CONFIRM_CONTROL_COUNT = 2
```

### 9.3 输出过流 OCP

快环软件 OCP：

```text
ocp_limit_ma = current_ref_ma + 1500mA
if ADC_Iout_positive > ocp_limit_ma:
    正常阶段立即 OCP
    预连接/AFE handover 阶段连续 3 次确认后 OCP
```

注意这里使用的是功率板输出电流 `ADC_Iout`，不是 BM2016 电池电流。这样可以保护功率板输出侧。

管理层也检查：

```text
硬件 FAULT_OC 有效
或 outputCurrentMa > current_limit_ma + 1500mA
```

### 9.4 输出最大电压边界

`BMS_POWER_OUTPUT_MAX_MV=40000` 是最终软件硬边界。即使自动充电目标是 37.8V，硬 OVP 计算也会被 40V 截住；数字电源目标也不能超过 40V。

## 10. 与硬件参数的匹配结论

### 10.1 已匹配

- R11/R12 `5mΩ` 与固件电流采样 `3.226mA/mV` 匹配。
- U1/U2 电流放大 `6.2k/100=62` 与固件注释和系数匹配。
- L2 `22uH` 与 `BMS_POWER_MAIN_INDUCTOR_UH=22` 匹配。
- U3/U4 `EG3112` 双半桥驱动与 HRTIMER 互补 PWM + 120ns 死区路径匹配。
- `37.8V/3A/0.3A` 与 9S 锂电 CC/CV 目标匹配。
- `18V/7A/92%` 在最低输入下可覆盖 `37.8V/3A`，计算上合理。

### 10.2 需要联调确认

- 电压采样：硬件分压从文本看约 `75k/4.7k`，固件标定用 `16.00`。如果实测显示值和万用表差异超过预期，需要重新标定 `BMS_ADC_VIN_GAIN_X1000/BMS_ADC_VOUT_GAIN_X1000`。
- 功率板文本里早期写 `4A/4.5A/90%`，但固件当前限制为 `3A/92%`。如果硬件最终要恢复 4A，需要同步重算输入限流、OCP 裕量、热降额和线束/保险丝余量。
- 当前 Buck 主路径没有使用 Buck 前馈公式，只用 PI 闭环。若后续追求更快动态响应，可以新增 Buck feedforward 作为 PI 初值或限幅参考，但不建议在未完成台架验证前直接替换闭环行为。

## 11. 联调检查点

上电不接电池、数字电源模式：

```text
1. Vin ADC 是否与输入电源一致。
2. Vout ADC 是否与万用表一致。
3. 低电压目标如 12V 时是否进入 Buck 区。
4. 高电压目标如 37.8V 时 24V 输入是否进入 Boost 区。
5. Boost duty 估算：24V -> 37.8V 时约 41.6%。
```

电子负载 CC：

```text
1. ADC_Iout 是否约等于电子负载读数。
2. 1A 时 ADC 放大后电压约 310mV。
3. 3A 时 ADC 放大后电压约 930mV。
4. OCP 触发点约为 current_ref + 1.5A。
```

接真实电池：

```text
1. 电池路径闭合前，反馈电流应来自 ADC_Iout。
2. 电池路径闭合后，正常 BMS 充电反馈电流应切到 BM2016 batteryCurrentMa。
3. CC 阶段目标电流为 3A，Vout 随电池电压上升。
4. packVoltage + 200mV >= 37.8V 后进入 CV。
5. CV 阶段 Vout 控制在 37.8V 附近，电池电流自然下降。
6. 电池电流 <= 300mA 且连续确认后进入 DONE。
```

保护验证：

```text
1. Vin 拉低到 18V 以下，应触发输入欠压或 duty guard。
2. Vout 超过 target+300mV 后，自动充电软 OVP 计数确认。
3. Vout 接近/超过 40V，应触发硬 OVP。
4. Iout 超过 current_ref+1.5A，应触发软件 OCP。
5. MOS/电感温度超过 50C 开始降额，超过 60C 停止输出。
```

## 12. 当前结论

按现有硬件网表/BOM和 `GD32G553_version_2_1` 固件看，Buck/Boost 相关核心参数已经基本对齐：

```text
Vin: 18V..60V
Vout: 自动充电 37.8V，数字电源/硬上限 40V
Iout: CC 3A，CV 截止 0.3A
PWM: 100kHz，5%..95%
ETA: 92%
L: 22uH
Rsense: 5mΩ
Current gain: 约 62
Current coefficient: 3.226mA/mV
```

最需要现场复核的是电压 ADC 标定系数，因为硬件分压文本和软件标定值之间存在“理论分压 vs 实测标定”的口径差异；电流采样、Boost duty、输入限流和 OVP/OCP 计算目前是自洽的。

## 1. 本轮联调背景

本轮联调围绕 9S 18650 锂电池充电展开。主要目标是让功率板在 BMS 模式下稳定完成预连接、闭合 AFE 主 FET，并进入真正的 CC/CV 充电。

早期现象：

- 数字电源模式效率可以到约 `80%~90%`。
- BMS 充电模式效率一度只有约 `10%`。
- CC 充电时直流稳压电源会出现过压或掉压。
- CV 模式目标电压达不到，例如电子负载设置 `36 V` 但 Vout 上不去。
- 预连接阶段 Vout 反复升降，常见现象是升到 `32 V` 左右后掉到 `10~11 V`，再重新升上去。
- 进入 CC 后功率级温度持续升高，最高曾到约 `70 C`。

关键判断方向：

- BMS 模式与数字电源模式的 Boost 驱动策略不一致。
- BMS 充电 CC 环路在电池这种电压源负载下可能把 Duty 推到不合理区域。
- 预连接阶段不应该长期承担供能，只应该做短时间电压对齐。
- 主 FET 闭合后，真正 CC 应使用 R71 / BM2016 电池侧电流作为权威反馈。

## 2. 已完成的主要工作

### 2.1 PWM 驱动与死区逻辑分析

做过的工作：

- 根据 EG3112 驱动芯片手册分析功率板 PWM 驱动逻辑。
- 检查主控内部 PWM 是否配置死区。
- 分析为什么 MCU 侧 PWM 有死区，但到 Gate2-L / Gate2-H 观测时像是没有死区。

阶段结论：

- MCU 侧互补 PWM 和死区配置需要满足驱动芯片输入逻辑。
- 如果驱动芯片内部或外部逻辑对高低侧信号做了重新整形，示波器在 Gate 输出端看到的波形不一定等同于 MCU 输出端死区。
- 后续判断死区应同时看 MCU PWM 引脚、驱动芯片输入脚、驱动芯片 Gate 输出脚，不能只看最终 Gate。

当前状态：

- PWM 死区问题已做过分析和代码尝试。
- 后续重点已转向预连接、AFE 闭合、CC 过温和效率问题。

### 2.2 自动充电逻辑分析与预检测逻辑调整

做过的工作：

- 分析自动模式下为何看起来像提前进入 CV。
- 讨论去掉原先“预检测 / 预连接确认过长”的逻辑。
- 将充电判断回归到电池包电压和单体最低电压：
  - 电池过低：涓流 / 预充。
  - 电池正常：CC。
  - 接近目标电压：CV。

阶段结论：

- 自动模式不能在主路径未闭合、电池电流未建立时就按 CV 逻辑长期调压。
- 预连接只用于电压对齐，不应替代 CC 充电。

### 2.3 PCHG / PDSG 与 CHG / DSG 路径控制

做过的工作：

- 结合网表和原理图分析主控板、BM2016、PCHG/PDSG/CHG/DSG 的连接关系。
- 确认当前工程中存在对应的 FET 控制函数。
- 明确当前工程是 `GD32G553_version_2/GD32G553`，不是其他历史文件夹。
- 对比过 `GD32G553_version_2_0615` 中曾经成功的 PCHG/PDSG 设置。
- 修正了 PCHG / PDSG 在存在故障时手动调试命令被拒绝的问题。

已解决的问题：

- CHG / DSG 能打开，但 PCHG / PDSG 打不开的问题已经阶段性解决。
- 用户反馈“现在测试通过了”，说明 PCHG/PDSG 控制链路已能按预期动作。

阶段结论：

- PACK、LD 引脚有电压并不等价于 PCHG/PDSG 已打开。
- PACK/LD 可能受体二极管、预充/预放路径、外部电源、测量参考点影响。
- 判断 FET 状态应以 BM2016 FET status、实际 Gate 状态和路径电流共同确认。

### 2.4 电流采样职责重新划分

做过的工作：

- 明确 R71 / BM2016、R11、R12 三类采样电阻职责。
- 根据用户补充，确认 R71 是 `5 mOhm`。
- 修改代码侧的充电电流使用逻辑，使电池侧电流成为 BMS 充电的权威反馈。

当前定义：

| 采样点 | 作用 |
| --- | --- |
| R71 / BM2016 | 电池侧权威充放电电流，用于真正 CC/CV 充电判断 |
| R11 输入电流 | 功率板输入保护、输入功率限制、效率保护 |
| R12 输出电流 | 功率板快速控制、输出过流保护、功率级局部保护 |

阶段结论：

- R12 在外部没有负载、主 FET 未闭合或电池电流尚未建立时，可能几乎没有有效电流。
- 不能用 R12 代替电池侧真实充电电流。
- 主 FET 闭合前不能期待有稳定的“充电电流”。

### 2.5 18650 充电参数调整

### 2.6 预连接目标与 AFE 闭合策略调整

早期尝试：

- 预连接目标曾设置为 `Pack + 300 mV`。
- 后来发现提高预连接目标后，过温现象更明显。
- 同时放宽部分上限后，功率级更容易在轻载 Boost 下吃掉输入功率。

最新目标：

- 预连接只做短时间电压对齐。
- Vout 阶梯式升压。
- 最高目标限制为满充电压 `37.8 V`。
- 当 `Vout >= Pack + 50 mV` 时，闭合 AFE 主 FET。
- 主 FET 闭合后进入真正 CC 充电。

最新完成代码调整：

- `BMS_PATH_PRECONNECT_TARGET_DELTA_MV` 改为 `50 mV`。
- `BMS_PATH_PRECONNECT_DELTA_MV` 改为 `50 mV`。
- `BMS_PATH_PRECONNECT_LOW_DELTA_MV` 改为 `50 mV`。
- 移除预连接 Boost 的 `600 ms` 运行、`1500 ms` 冷却节拍。
- 预连接目标改为按 `500 mV` 步进阶梯上升。
- 预连接最终上限钳位到 `37.8 V`。
- READY 条件改成 `Vout >= Pack + 50 mV`。
- 预连接期间禁用 stall recover，避免 PWM 被误复位。
- 预连接 Boost Duty 上限放开到全局 PWM 上限。

涉及文件：

- `Driver/bms_board_config.h`
- `Driver/charge_manager.c`
- `Driver/power_path_manager.c`
- `Driver/power_control.c`

## 4. 现有问题与待验证项

### 4.1 CC 充电仍有过温风险

现象：

- 进入 CC 后一段时间功率级温度持续上升。
- 曾出现最高约 `70 C`。
- 典型功率不匹配现象：

```text
输入：24 V * 0.68 A ≈ 16 W
电池：35 V * 0.2 A ≈ 7 W
```

判断：

- 大量能量可能消耗在功率级。
- 可能与轻载 Boost、同步整流反向电流、Duty 不合理、闭合瞬态、采样反馈选择有关。
- 这不是单纯提高 Vout 目标能解决的问题。

当前状态：

- 最新预连接逻辑减少了“闭合前长时间 Boost 供能”的可能性。
- 但过温是否解决，需要上板验证。

### 4.2 预连接 Vout 是否能跨过 Pack + 50 mV

现象：

- 修改前 Vout 常卡在 `32 V` 左右，然后掉到 `10~11 V`。

当前状态：

- 已从代码上去掉两个明显会导致掉电的软件因素。
- 仍需要实测确认 Vout 是否能稳定升到 Pack 附近并触发 READY。

验证重点：

- Power Debug 中 `preconnectFlags` 是否经历 ACTIVE -> READY。
- `preconnectReason` 是否变成 OK。
- CHG/DSG 是否闭合。
- 闭合后是否进入 CC。

### 4.3 CC 后效率低的问题仍需继续定位

现象：

- 输入功率明显大于电池吸收功率。
- 充电电流可能只有 `0.02 A~0.2 A`，但输入电流仍然较高。

后续建议：

- 同时记录 R11 输入电流、R12 输出电流、R71 电池电流。
- 计算：

```text
Pin = Vin_power_board * Iin_R11
Pbat = Pack * Ibat_R71
```

- 若 `Pin >> Pbat` 持续存在，应立刻降 Duty 或进入轻载保护，而不是继续推 Boost。

### 4.4 R12 输出电流在预连接 / 无负载时不适合做 CC 依据

现象：

- 外部没有负载时，R12 上电流几乎没有。
- 预连接阶段主 FET 未闭合，电池侧真实电流也不会稳定建立。

结论：

- 预连接阶段不能以“没有充电电流”为异常。
- 主 FET 闭合后，才用 R71 / BM2016 做真正 CC。

### 4.5 直流稳压电源过压 / 掉压仍需结合瞬态分析

可能原因：

- Vout 未高于 Pack 时闭合路径，电池能量可能瞬态反灌。
- 同步 Boost 轻载时电感电流过零后可能短暂反向。
- AFE 主 FET 打开瞬间，电池包钳位 Vout，产生瞬态冲击。
- 输入电源限流后，Vin 下跌，控制环继续推 Duty，造成功率级异常工作。

当前处理：

- 预连接目标改为 Vout 高于 Pack `50 mV` 后再闭合。
- 主 FET 闭合后才进入 CC。

仍需验证：

- 闭合瞬间是否仍有反向电流。
- 输入电源是否还会报过压。
- 是否需要更细的 AFE handover 限 Duty / 限斜率策略。

### 4.6 BM2016 单体采样异常仍需单独处理

现象：

- `cell4 = 655271` 明显像浮空随机数据。
- `cell5` 像读到了两节电池电压之和。
- 用户确认硬件电池本身电压正常。

判断：

- 这更像 AFE 采样线、连接顺序、通道配置或解析映射问题。
- 不应和功率板 Boost 过温问题混在一起判断。

当前状态：

- 本轮重点在充电路径和功率控制。
- 单体采样异常仍需后续单独检查。

## 5. 当前建议的下一步测试顺序

### 5.1 先只验证预连接

条件：

- 直流稳压电源建议先限流保守设置。
- 观察 Vout、Pack、PCHG/PDSG、CHG/DSG、preconnectFlags。

判定：

### 5.2 再验证 AFE 闭合瞬间

重点：

- `Vout >= Pack + 50 mV` 后 CHG/DSG 是否闭合。
- 闭合瞬间输入电源是否报过压。
- 是否有输出过流。
- Duty 是否突变。

### 5.3 最后验证 CC

重点：

- R71 / BM2016 电池电流是否建立。
- CC 是否能稳定到 `600~1000 mA` 范围。
- 输入功率与电池功率是否接近。
- 温度是否继续快速上升。

# GD32G553_version_2 预充路径 FET 配置错误修复

日期：2026-06-18

工程范围：
- 固件：`G553/GD32G553_version_2/GD32G553`
- 硬件：主控板原理图第5页（电芯充放电路径）
- 主要文件：
  - `Driver/power_path_manager.c`
  - `Driver/charge_manager.c`
  - `Driver/bms_board_config.h`

## 1. 问题现象

### 初始现象
- CC 模式下 Vout 过压，触发硬件 OVP 锁存（42.8V）
- 预连接阶段 Vout 升到 32.556V 后长时间不动
- 预连接一直卡在 PRECHECK 状态，无法完成主 FET 闭合
- 修改参数后问题反复出现，无法稳定进入充电

### 关键调试数据
1. **第一次过压触发时：**
   - Vout = 42.97V（超硬限 42.8V）
   - 触发 Duty = 43.53%
   - 预连接原因：BM2016主FET合闸失败
   - Vout-Stack 压差 > 100mV（超芯片门闩）

2. **预连接卡死时：**
   - Vout = 32.556V（目标 Pack = 34.6V）
   - Duty = 2.00%（coast 停机）
   - Iin ≈ 0A（几乎无输入电流）
   - 预连接 Flags: 0x0A (ACTIVE/VOUT_LOW)

3. **参数调整后仍过冲：**
   - 电流限制从 600mA → 150mA → 100mA
   - Coast 阈值从 100mV → 500mV → 1000mV
   - 目标从 Pack → Pack-20 → Pack-300
   - **问题依然存在**

## 2. 排查过程

### 2.1 初步假设：Boost 参数问题

**尝试的修复：**
1. 降低预连接电流限制（600mA → 100mA）
2. 提高 coast 阈值（100mV → 1000mV）
3. 降低预连接目标（Pack → Pack-300mV）
4. 删除各种 guard 机制
5. 修改 READY 判据（对称窗口 → 下沿触发）
6. 加入 FET 闭合确认（压差 < 50mV）

**结果：** 问题依然存在，说明不是 Boost 参数问题。

### 2.2 深入分析：硬件拓扑

**查看硬件原理图和网表：**

从 `hardware/主控板_text.txt` 第5页发现：
```
预充电：Q2开，Q5开
预放电：Q3开，Q4开
充电：Q4开，Q5开
```

**关键发现：预充需要 Q2(PCHG) + Q5(DSG)，不是 Q2 + Q3！**

### 2.3 代码分析：FET 配置错误

**查看代码配置：**
```c
// bms_board_config.h
#define BMS_PATH_HARDWARE_PRECONNECT_USE_PCHG 1U  // 使能 PCHG
#define BMS_PATH_HARDWARE_PRECONNECT_USE_PDSG 1U  // 使能 PDSG ← 错误！

// power_path_manager.c
#define POWER_PATH_HARDWARE_PRECONNECT_FET_MASK (PCHG | PDSG)
```

**代码配置的预充路径：PCHG + PDSG（错误）**

**硬件实际的预充路径：PCHG + DSG（正确）**

## 3. 硬件拓扑分析

### 3.1 完整电路拓扑

```
功率板VOUT+ (VIN+)
    ↓
    ├─→ [预充电阻 R24~R34 并联 = 187.5Ω] → [Q2 PCHG FET] ──┐
    │                                                        ├─→ bat+
    └─→ [Q4 CHG FET 直连无电阻] ───────────────────────────┘
    
bat+ → [电池包 9S: 27~37.8V] → bat-
    
bat-
    ├─→ [Q3 PDSG FET] ──────────────┐
    │                                ├─→ 功率板VOUT- (VIN-)
    └─→ [Q5及Q21~Q23 DSG FET多个并联] ─┘
```

### 3.2 三种工作模式

#### 模式1：预充电（PCHG + DSG）
```
VIN+ → 187.5Ω → Q2(闭合) → bat+ → 电池 → bat- → Q5(闭合) → VIN-
      ↑                                              ↑
   预充电阻限流                                  主放电FET
   
Q3(PDSG): 断开
Q4(CHG): 断开
```

**作用：**
- 通过 187.5Ω 预充电阻给电池充电，限流保护
- 用主放电 FET (Q5) 回流，大电流能力

#### 模式2：充电（CHG + DSG）
```
VIN+ → Q4(闭合) → bat+ → 电池 → bat- → Q5(闭合) → VIN-
      ↑                                      ↑
   直连无电阻                              主放电FET
   
Q2(PCHG): 断开
Q3(PDSG): 断开
```

**作用：**
- 大电流直接充电，无电阻损耗

#### 模式3：预放电（PDSG + CHG）
```
VIN+ → Q4(闭合) → bat+ → 电池 → bat- → Q3(闭合) → VIN-
                                      ↑
                                  预放电FET
Q2(PCHG): 断开
Q5(DSG): 断开
```

**作用：**
- 主要用于放电场景，通过预放电路径限流

### 3.3 为什么必须用 DSG 而不是 PDSG？

#### 错误配置（PCHG + PDSG）的问题：

1. **PDSG 是小功率 FET**
   - 设计用于预放电场景，可能只能持续通过 1~2A
   - 预充时 Boost 输出电流需要 150~200mA，PDSG 可能过热

2. **导通阻抗大**
   - PDSG 可能 Rds_on ≈ 100mΩ
   - DSG 多个并联后 Rds_on ≈ 10mΩ（低10倍）

3. **电流能力不足**
   - PDSG 过热后阻抗上升或进入限流保护
   - Boost 看到的负载阻抗变化剧烈
   - PI 控制器无法稳定，过冲严重

4. **导致现象：**
   - Vout 慢慢爬升到 32~33V 后卡住（PDSG 电流能力不足）
   - 无法推到 READY 阈值（34.55V）
   - 或者过冲失控（负载阻抗不稳定）

#### 正确配置（PCHG + DSG）的优势：

1. **DSG 是大功率 FET**
   - 设计用于正常充放电，可以持续通过 10A+
   - 多个 FET 并联（Q5, Q21, Q22, Q23）
   - 散热好，不会过热

2. **阻抗稳定**
   - DSG 阻抗稳定在低值
   - Boost 负载特性稳定
   - PI 控制器容易收敛

3. **电流能力充足**
   - 可以支持预充需要的 100~200mA 输出电流
   - Vout 可以顺利推到 Pack 附近

## 4. 根本原因分析

### 4.1 为什么会有这个错误？

可能原因：
1. **代码移植问题**：从其他 BMS 项目移植，那个项目的预充设计不同
2. **理解偏差**：编写时误以为"预充"就应该配"预放"
3. **测试覆盖不全**：特定条件下（不同输入/电池电压）可能能工作，掩盖了问题

### 4.2 为什么参数调整无效？

**所有参数调整都在试图解决一个错误前提下的问题：**
- 前提错误：预充回路使用 PDSG（小功率）
- 调整参数：降低电流、提高 coast 阈值、降低目标
- 结果：治标不治本，因为回路本身就错了

**真正的问题：**
- 不是 Boost 参数问题
- 不是 PI 调节问题
- 不是目标设定问题
- **而是 FET 配置错误，使用了错误的回路**

## 5. 修复方案

### 5.1 代码修改

#### 修改1：bms_board_config.h

```c
/*
 * 预充路径配置：PCHG + DSG（不是 PDSG）
 * 硬件拓扑：VIN+ → 187.5Ω → PCHG → bat+ → 电池 → bat- → DSG → VIN-
 * 预充需要预充电阻(PCHG路径)限流 + 主放电FET(DSG)提供回路。
 * PDSG 是预放电用，不参与预充。
 */
#define BMS_PATH_HARDWARE_PRECONNECT_ENABLE   1U
#define BMS_PATH_HARDWARE_PRECONNECT_USE_PCHG 1U
#define BMS_PATH_HARDWARE_PRECONNECT_USE_PDSG 0U  // 改为 0，不用 PDSG
#define BMS_PATH_HARDWARE_PRECONNECT_USE_DSG  1U  // 新增，预充需要 DSG 回路
```

#### 修改2：power_path_manager.c

```c
#if BMS_PATH_HARDWARE_PRECONNECT_USE_PCHG
#define POWER_PATH_HARDWARE_PRECONNECT_PCHG_MASK AFE_GD30BM2016_FET_STATUS_PCHG
#else
#define POWER_PATH_HARDWARE_PRECONNECT_PCHG_MASK 0U
#endif

#if BMS_PATH_HARDWARE_PRECONNECT_USE_PDSG
#define POWER_PATH_HARDWARE_PRECONNECT_PDSG_MASK AFE_GD30BM2016_FET_STATUS_PDSG
#else
#define POWER_PATH_HARDWARE_PRECONNECT_PDSG_MASK 0U
#endif

#if BMS_PATH_HARDWARE_PRECONNECT_USE_DSG
#define POWER_PATH_HARDWARE_PRECONNECT_DSG_MASK AFE_GD30BM2016_FET_STATUS_DSG
#else
#define POWER_PATH_HARDWARE_PRECONNECT_DSG_MASK 0U
#endif

#define POWER_PATH_HARDWARE_PRECONNECT_FET_MASK (POWER_PATH_HARDWARE_PRECONNECT_PCHG_MASK | \
                                                 POWER_PATH_HARDWARE_PRECONNECT_PDSG_MASK | \
                                                 POWER_PATH_HARDWARE_PRECONNECT_DSG_MASK)
```

### 5.2 其他保留的修复（之前调整的有效部分）

以下修复虽然不是根本原因，但也是合理的改进，保留：

1. **预连接目标：Pack - 300mV**
   - 让 Boost 推到略低于 Pack，靠过冲接近 Pack
   - 配合 coast 阈值控制过冲幅度

2. **电流限制：100mA**
   - 小电流慢速充电容，减少过冲

3. **Coast 阈值：1000mV**
   - 给过冲留足余量，不会过早停机

4. **READY 判据：Vout >= Pack - 50mV**
   - 下沿触发，提前进入 READY

5. **FET 闭合确认：Vout-Stack < 50mV**
   - 防止提前切 CC 导致目标跳变过冲

6. **Handover headroom：0%**
   - 避免 CC 切换时 duty 过高

## 6. 预期效果

### 修复前（PCHG + PDSG）

```
1. Boost 启动，目标 Pack-300 (34.3V)
2. Vout 慢慢爬升到 32~33V 左右
3. PDSG 电流能力或散热不足，阻抗上升
4. Vout 卡住不动或缓慢爬升
5. 永远到不了 READY 阈值（Pack-50 = 34.55V）
```

### 修复后（PCHG + DSG）

```
1. Boost 启动，目标 Pack-300 (34.3V)
2. DSG 大电流回路形成，Boost 顺利推动
3. Vout 稳定爬升到 34.3V 附近
4. 可能过冲到 34.5~35V（在 1000mV coast 阈值内）
5. Vout 达到 READY 阈值（34.55V）
6. 压差收敛到 < 50mV，主 FET 闭合
7. 切换到 CC 充电，Boost 目标升到 37.8V
8. 正常 CC/CV 充电流程
```

## 7. 关键经验总结

### 7.1 硬件理解的重要性

**教训：**
- 代码逻辑必须严格匹配硬件设计
- 不能想当然地认为"预充配预放"
- 必须查看原理图确认实际连接

**方法：**
- 看原理图文字标注（"预充电：Q2开，Q5开"）
- 看网表连接关系
- 理解每个 FET 的设计用途

### 7.2 排查方向的重要性

**错误的排查方向：**
- 一直在调 Boost 参数（电流、coast、目标）
- 以为是控制算法问题
- 以为是 PI 参数问题

**正确的排查方向：**
- 先确认硬件连接是否正确
- 再确认代码配置是否匹配硬件
- 最后才调控制参数

### 7.3 现象与本质的区别

**现象：**
- Vout 过冲
- Vout 卡死
- 预连接无法完成

**表面原因：**
- Boost 电流太大
- Coast 阈值太低
- 目标设定不合理

**本质原因：**
- FET 配置错误，使用了错误的回路
- PDSG 电流能力不足，无法支持预充

**启示：**
- 不要只看现象调参数
- 要深入分析硬件拓扑
- 找到根本原因才能彻底解决

## 8. 后续测试中发现的问题

### 8.1 问题A：电流倒灌（输入电压过低）

#### 现象
- 电源设置：24V, 3A 恒流模式
- 实际显示：32.7V, 0.00A（变成恒压模式）
- 电池电流：-0.094A（放电/倒灌）
- 预连接完成，主 CHG+DSG 闭合
- 进入 CC 模式，但 Vout 不上升

#### 原因分析

**倒灌路径：**
```
电池 34.6V → CHG+DSG(闭合) → Vout 34.5V → Boost(未工作) → Vin 32.7V → 电源 24V
                                                                    ↑
                                              电池反向把电源电压拉高到 32.7V
```

**为什么会倒灌：**
1. **预连接完成**，主 FET (CHG+DSG) 闭合
2. **电池电压 34.6V > 电源设定 24V**
3. **Boost 从 32.7V 升到 37.8V 需要大升压比**，但电源只能提供很小电流
4. **电源被倒灌拉到 32.7V 后进入 CV 模式**，输出 0A
5. **Boost 无法正常工作**，形成死锁

**为什么预连接能完成但 CC 充电失败：**
- 预连接目标：Pack-300 = 34.3V，升压比 34.3/32.7 = 1.05（小）✓
- CC 充电目标：37.8V，升压比 37.8/32.7 = 1.156（大），且电源无法提供足够电流 ✗

#### 解决方案

**提高电源电压到 36V：**
- 电源：36V, 3A 恒流模式
- 36V > 电池 34.6V，不会倒灌
- Boost 从 36V 升到 37.8V，升压比 1.05（小，容易稳定）
- 电流方向正确：电源 → Boost → 电池

**关键经验：**
- **测试 BMS 充电时，电源电压必须高于电池满电电压**
- 对于 9S 锂电池（满电 37.8V），建议电源电压 **36~40V**
- 不能用 24V 测试全流程（只能测预连接阶段）

---

### 8.2 问题B：Buck/Boost 模式选择错误（预连接目标过低）

#### 现象（36V 输入时）
- 电源：36V, 3A 恒流模式（两通道串联）
- 充电模式：CV
- 功率模式：**Buck（降压）**，不是 Boost！
- Duty = 2.00%（Buck duty）
- **Boost Low Duty = 0.00%（Boost 完全停止）**
- Vout = 0.044V（几乎为 0）
- 预连接 Flags: ACTIVE/VOUT_LOW
- 预连接原因：Vout低于Stack，软启动控开

#### 原因分析

**模式选择逻辑：**
```c
// power_control.c line 1448-1451
if((uint32_t)input_voltage_mv > region_target_mv + POWER_REGION_MARGIN_MV) {
    s_power.powerStageMode = POWER_STAGE_MODE_BUCK;  // 强制 Buck
    boost_duty = 0U;  // Boost 停止
}
```

**触发条件：**
- Vin = 36.042V
- 预连接目标 = Pack - 300 = 34.228V
- **36V > 34.228V + margin → 进入 Buck 模式**

**问题：**
1. **硬件是纯 Boost 拓扑**，没有 Buck 降压到电池的能力
2. **Buck 模式下 Boost 半桥停止工作**（boost_duty = 0）
3. **Vout 输出为 0，无法预连接**

**为什么会这样：**
- 我们为了解决过冲问题，把预连接目标设为 **Pack - 300mV**
- 在输入 24V 时：目标 34.3V > Vin 24V → 用 Boost 升压 ✓
- 在输入 36V 时：目标 34.3V < Vin 36V → 用 Buck 降压 ✗

#### 解决方案

**修改预连接目标：Pack + 100mV（不是 Pack - 300mV）**

```c
// charge_manager.c 预连接目标计算
target_mv = (uint32_t)afe->packVoltageMv + 100U;  // Pack + 100mV
```

**修改后的行为：**
- 预连接目标 = Pack + 100 = 34.7V
- Vin = 36V，目标 34.7V，差距 1.3V（很小）
- 进入 **Buck-Boost 过渡区或 Boost 区**，不会强制 Buck
- Boost 正常工作，Vout 可以爬升

**为什么 Pack + 100mV 更好：**

1. **确保 Boost 模式**
   - 目标接近 Vin，不会触发强制 Buck

2. **预充电阻形成负载**
   - Vout 34.7V > Pack 34.6V
   - 预充电阻开始充电：(Vout - Pack) / 187.5Ω ≈ 0.5mA
   - 形成负载，帮助 Boost 稳定

3. **满足芯片门闩**
   - 压差 100mV，在 BM2016 门闩范围内

4. **避免过冲**
   - 目标已经接近 Pack，过冲幅度小
   - Coast 阈值 1000mV 留足余量

**关键经验：**
- **预连接目标不能设太低**（Pack - 300 太低）
- 目标应该 **略高于或等于 Pack**（Pack + 100 或 Pack）
- 确保在任何合理的输入电压下（24V~40V）都使用 Boost 模式

```
   - 预充电阻开始给电池充电，形成负载
   
4. 主 FET 闭合
   - 压差 < 50mV，软件确认
   - CHG + DSG 闭合，关闭 PCHG
   - batteryPathEnabled = 1
   
5. CC 充电
   - Boost 目标切到 37.8V
   - Vin 36V → 37.8V，升压比 1.05，稳定工作
   - 开始 CC 充电 600mA
   
6. CV 充电
   - Pack 电压达到 37.8V
   - 电流自然下降
   - 降到 100mA 截止
```

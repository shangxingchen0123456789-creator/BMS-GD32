## 0.1 上板复测结论：低阈值同步整流方案已回退

2026-06-19 上板验证结果：把 CC 异步整流进入/退出阈值从 `300mA/500mA` 改为 `40mA/80mA` 后，未解决 CC 电流卡死，反而出现**电流倒灌进直流稳压电源**的危险现象。

因此该方案不能继续作为修复方案使用。当前代码已经回退到原始 `300mA/500mA` 逻辑，并重新编译通过：

```text
codex_build_async_revert.log:
"..\Output\GD32G553_CHG.axf" - 0 Error(s), 0 Warning(s).
```

后续诊断方向应从“简单降低异步整流阈值”转为：

1. 确认四开关 Buck-Boost 同步整流时序是否允许电池侧能量经电感回灌到输入侧。
2. 确认 BM2016 电池电流反馈与功率板 `ADC_Iout` 是否存在方向/采样延迟问题。
3. 在没有逐周期电流过零检测或明确反向电流保护前，不应在 CC 阶段简单强制 Q2 同步整流。

## 0.2 本轮澄清：直流源读数是 0.38A，不是 0.38V

2026-06-19 复核截图时确认，直流稳压电源读数为约 `0.38A`，不是输入电压 `0.38V`。因此当前现象不是输入端塌压或电源被短路，而是固件 `Iin≈0.48A` 相对直流源均值约偏高 `0.10A`。

这个偏差不足以触发原始输入保护：`POWER_INPUT_GUARD_IIN_MAX_MA = 1800mA`。所以本轮不应通过放宽 input guard 来解释或修复电流卡死。此前临时把输入保护阈值放到 `BMS_DEFAULT_INPUT_CURRENT_LIMIT_MA=6500mA` 并加 8 次确认的改动已经撤回，恢复为 `1800mA` 立即降 duty。

## 0.3 当前采用的安全修正

当前代码不再提前打开 Q2 同步整流，仍保持 CC 低电流 Boost 异步整流阈值为 `300mA/500mA`。新的尝试是只在 **CC 且当前处于异步 Boost 整流** 时，给 duty 上限计算额外增加裕量，补偿 Q2 体二极管压降造成的升压余量损失。

`800mV` 版本上板后，充电电流从约 `0.12A` 上升到约 `0.20A`，直流稳压电源显示约 `24V/0.58A`。这说明方向有效，但裕量仍不足。当前已升级到 `1600mV`：

```c
#define POWER_CC_ASYNC_BOOST_EXTRA_MARGIN_MV   1600U
```

该裕量不改变 Q2 同步整流策略。已编译验证：

上板复测重点：

1. Power Debug 中 Boost 仍应显示异步，且直流源不能再出现反向电流。
2. Duty 不应继续精确卡在约 `32.3%`；若仍卡住，要看 `targetVoltageMv` 是否被上层限制在约 `35.4V`。
3. 若 `targetVoltageMv=37.8V` 且 1600mV 版本电流仍上不去，可再小步提高 `POWER_CC_ASYNC_BOOST_EXTRA_MARGIN_MV`，例如 `2000mV`，但每步都必须保留 OVP/OCP 观察。

## 0.4 1600mV 版本上板结果：先确认是否进入涓流/限流

`1600mV` 版本上板后，充电电流约 `0.215A`，直流稳压电源约 `24V/0.59A`。相比 `800mV` 版本的约 `0.20A`，增幅很小。

这个电流值非常接近固件涓流电流 `TRICKLE_CURRENT_MA = 200mA`，因此下一步不要先继续盲目提高 Boost 裕量，而应优先确认 Power Debug 中的：

```text
chargeState / chargeMode
targetCurrentMa
softCurrentMa
limitedCurrentMa / inputLimitedCurrentMa
dutyX100
targetVoltageMv
```

判断规则：

1. 若 `targetCurrentMa` 或 `softCurrentMa` 约为 `200mA`，说明当前不是 CC 0.6A 卡死，而是上层状态机处于 TRICKLE 或 DPM/限流；需要回查 `cellMinMv`、`chargeState`、`Power_Manager_Limit_Current()`。
2. 若 `targetCurrentMa=600mA` 且 `softCurrentMa` 已爬到 `600mA`，但电池电流仍只有 `0.215A`，再继续看 duty 是否被 Boost duty 上限钳住。
3. 若 duty 已接近当前上限但电流仍不上去，再考虑把 `POWER_CC_ASYNC_BOOST_EXTRA_MARGIN_MV` 提到 `2000mV`，或直接让 CC Boost duty 上限使用最终 `targetVoltageMv` 计算。

## 0.5 参数与 Duty 复核：36.1% 是最终目标理想占空比

上位机下发参数确认：

```text
目标电压: 37.80V
目标电流: 0.60A
单体最低: 3.810V
```

因此不是参数下发错误，也不是应进入涓流。现场 Duty 一直在约 `36.1%`，这个值与 `24V -> 37.8V` 的理想 Boost 占空比一致：

```text
Dideal = (37.8 - 24.0) / 37.8 = 36.5%
```

这说明功率环被“理想 Boost duty”钳住了，没有给异步整流体二极管、线路压降、电感 DCM 纹波留额外占空比。因此当前修正改为：

1. CC 模式的 Boost duty 上限直接按最终 `targetVoltageMv` 计算。
2. `+6%` duty headroom 版本上板后，充电电流升到约 `0.4A`，直流稳压电源约 `24V/1.25A`（约 `30W`），证明 duty headroom 方向有效。
3. 为避免电流超过 `500mA` 后退出异步并重新打开 Q2 同步整流，当前版本在 CC Boost 阶段固定保持异步。
4. 当前进一步把异步 Boost duty headroom 提到 `+10%`。
5. 仍保留 `BMS_POWER_BATTERY_BOOST_DUTY_MAX_X100 = 5200`、OVP/OCP、input guard。

```c
#define POWER_CC_ASYNC_BOOST_DUTY_HEADROOM_X100 1000U
```

## 0. 一句话结论

原始假设：CC 充电时电流被锁死在 ~0.12A（目标 0.6A），根因**不是效率问题**，而是**输出侧同步整流管 Q2 在 CC 模式下被错误地长期关闭（走体二极管），二极管 0.7V 压降吃掉了 Boost 的升压裕量，导致电流灌不进电池**；电流上不去又让退出条件永远不满足，形成自锁。

上板结果修正：这个假设只解释了“异步时电流低”的一部分现象，但“降低 CC 异步阈值、让 Q2 尽早同步整流”会引入电流倒灌风险，已经被实测否定。后续不能直接采用 40mA/80mA 阈值方案。

## 2. 先排除"效率低"这个误判

按面板第一眼算：输入 23.7V×（虚高的输入电流）≈ 二十几 W，输出 4W，像是效率 < 10%。**但这是假象**，证据：

- **MOS 34.8°C、电感 33.9°C，基本是室温。** 若真有 ~20W 损耗砸在功率级，这些器件早该七八十度。器件是凉的 → 功率级并没有耗散那么多功率 → 输入电流是虚读（轻载 + 异步整流下电感电流断续，ADC 同步采样点落在纹波上，把均值读大了）。

所以真正要解决的不是"效率"，而是**充电电流为什么卡在 0.12A**。

## 3. 关键器件 Q2 是什么

### 3.1 功率板拓扑（四开关同步 Buck-Boost）

```
            L2 (22µH)
VIN+ ──┬── Q1(H) ──┬─HS-1───[ L2 ]───HS-2─┬── Q2(H) ──┬── VOUT+
       │           │                       │           │
       │          Q3(L)                   Q4(L)         │
      Cin          │                       │           Cout
       │          PGND                    PGND          │
VIN- ──┴───────────────────────────────────────────────┴── VOUT-
       输入侧半桥                          输出侧半桥
       (Buck 半桥)                         (Boost 半桥)
```

各管角色（依据 `功率板网表.tel`）：

| 器件 | 位置 | 栅极信号 | 网表依据 |
|---|---|---|---|
| Q1 | 输入侧**高边** | PWM1H / Gate1-H | `Gate1-H; Q1.4`，`VIN+; Q1.5-9` |
| Q3 | 输入侧**低边** | PWM1L / Gate1-L | `Gate1-L; Q3.4`，`PGND; Q3.1-3` |
| **Q2** | **输出侧高边** | **PWM2H / Gate2-H** | `Gate2-H; Q2.4`，`VOUT+; Q2.5-9`，`HS-2; Q2.1-3` |
| Q4 | 输出侧低边 | PWM2L / Gate2-L | `Gate2-L; Q4.4`，`PGND; Q4.1-3` |

- Q1~Q4 均为 **CJAC80SN10**（PDFN-8 封装 N-MOS，功率板 BOM 已确认）。
- 输出侧半桥由 **U4/EG3112** 驱动（SOIC-8 半桥驱动，U4.6 接 HS-2 节点，确认为输出侧）。
- 主电感 **L2 = 22µH**（IND-TH，直插）。
- 输出电流采样电阻 **R12 = 5mΩ**（2512）在 VOUT- 侧；R11 同为 5mΩ 在输入侧。
- 以上封装/阻值/连接均已对照 `hardware/功率板网表.tel` 和 `hardware/功率板bom.xlsx` 核实。

### 3.2 Boost 工作时 Q2 的作用 = 同步整流管

Boost 升压（能量从 VIN 侧搬到 VOUT 侧）时，输出侧半桥这样配合：

- **Q4（低边，主开关）** 导通 → 电感 L2 经 Q4 对地储能；
- **Q4 关断** → 电感电流必须继续流，经**输出侧高边 Q2 续流到 VOUT+**，把能量送进电池。

Q2 在 Q4 关断期间承担续流。它有两种工作方式：

1. **同步整流（正确、低损耗）**：MCU 主动驱动 Q2 导通，电流走 Q2 的**沟道**（导通压降 = I × Rds_on，毫伏级）。
2. **异步整流（高损耗）**：MCU 不驱动 Q2，电流只能走 Q2 的**体二极管**，压降固定 ~0.7V。

代码里关闭 Q2 主动驱动的就是这一行：
```c
// power_control.c  Power_Pwm_Active_Output_Channels()
channels &= ~((uint32_t)HRTIMER_ST1_CH0);   // ST1_CH0 = PWM2H = Q2 高边
```
（`HRTIMER_SLAVE_TIMER1` 对应 PWM2H/PWM2L，见 [power_control.c:318](Driver/power_control.c#L318)、[585](Driver/power_control.c#L585)；调用约定 `(timer, channel_h, channel_l)` 决定 CH0=高边。）

## 4. 为什么会卡死（自锁机理）

### 4.1 路径阻抗与升压裕量

用正确的充电电流 0.12A 算：
```
路径阻抗 ≈ (Vout − Pack) / I = 172mV / 0.12A ≈ 1.4 Ω
（这是电池内阻、连接器、BM2016 主 FET、采样电阻、走线等的综合等效阻抗；只能说明当前压差与电流量级自洽，不能单独证明或排除某一处高阻故障）
```
要把电流抬到 0.6A，需要约 `0.6A × 1.4Ω ≈ 0.86V` 的过驱。当前 CC/涓流下 Boost 区域/占空比限制参考值由 [Power_Control_Cc_Duty_Target_Mv](Driver/power_control.c#L1130) 计算：
```
region_target = min(max(packVoltage + 800mV, Vout + 800mV), targetVoltage)
```
也就是说代码本来会给 pack/Vout 上方留约 800mV 的升压裕量，接近这次 0.6A 所需的过驱量。

### 4.2 异步整流把裕量吃光

但 CC 模式下 Q2 走了体二极管，多出 ~0.7V 压降：
```
净过驱 ≈ 800mV(软件预留裕量) − 700mV(体二极管) ≈ 100mV
```
只剩约 100~170mV 净过驱（实测 Vout−Pack=172mV 正好吻合）→ 电流只能灌 **~0.12A**。

### 4.3 自锁闭环

看 CC 模式的异步整流判定（[Power_Control_Async_Boost_Should_Run](Driver/power_control.c#L1398)）：

```c
#if BMS_POWER_LIGHT_LOAD_ASYNC_BOOST_RECTIFIER
    if(s_power.mode == (uint8_t)BMS_CHARGE_MODE_CC) {
        if(s_async_boost_rectifier != 0U) {
            return (measured_current_ma <= POWER_ASYNC_BOOST_EXIT_CURRENT_MA) ? 1U : 0U;  // 退出阈值 500mA
        }
        return (measured_current_ma <= POWER_ASYNC_BOOST_CURRENT_MAX_MA) ? 1U : 0U;        // 进入阈值 300mA
    }
```
- 进入异步：`measured ≤ 300mA`（`POWER_ASYNC_BOOST_CURRENT_MAX_MA`）
- 退出异步：`measured > 500mA`（`POWER_ASYNC_BOOST_EXIT_CURRENT_MA`）

软启动从 0 爬升 → 一开始电流 < 300mA → **进入异步** → Q2 走体二极管 → 裕量被吃掉 → 电流卡在 0.12A → **0.12A 永远 < 500mA 退出阈值 → 永远退不出异步** → 永远建不起压 → 永远 0.12A。**死锁。**

### 4.4 效率为什么显示低

电流被锁在 0.12A → 输出仅 35V×0.12A ≈ 4.2W。而板子固定损耗（输入侧 200kHz 开关、栅驱动、磁芯）是恒定几瓦，不随输出变 → 占比过大 → 效率被压到 40% 多。**把电流解锁到 0.6A（输出 ~21W），同样的固定损耗，效率自然回到 80%+。**

## 5. 应该改什么

**目标：让 CC 正常充电工况走同步整流（Q2 主动导通），只在极轻载时才保留异步兜底。**

### 改动 1：新增 CC 专用阈值（power_control.c 宏定义区，约 45~46 行附近）

在
```c
#define POWER_ASYNC_BOOST_CURRENT_MAX_MA       300U
#define POWER_ASYNC_BOOST_EXIT_CURRENT_MA      500U
```
后面增加：
```c
/*
 * CC 充电稳态应走同步整流（输出侧高边 Q2 主动导通），避免体二极管 0.7V
 * 压降吃掉 Boost 升压裕量导致电流灌不进电池。仅在 CC 极轻载或软启动
 * 初期保留异步整流兜底，防止同步整流下电感反向电流倒灌。
 */
#define POWER_CC_ASYNC_BOOST_ENTER_MA          40U
#define POWER_CC_ASYNC_BOOST_EXIT_MA           80U
```

### 改动 2：CC 分支改用低阈值（[Power_Control_Async_Boost_Should_Run](Driver/power_control.c#L1398)）

把
```c
    if(s_power.mode == (uint8_t)BMS_CHARGE_MODE_CC) {
        if(s_async_boost_rectifier != 0U) {
            return (measured_current_ma <= POWER_ASYNC_BOOST_EXIT_CURRENT_MA) ? 1U : 0U;
        }
        return (measured_current_ma <= POWER_ASYNC_BOOST_CURRENT_MAX_MA) ? 1U : 0U;
    }
```
改为
```c
    if(s_power.mode == (uint8_t)BMS_CHARGE_MODE_CC) {
        if(s_async_boost_rectifier != 0U) {
            return (measured_current_ma <= POWER_CC_ASYNC_BOOST_EXIT_MA) ? 1U : 0U;
        }
        return (measured_current_ma <= POWER_CC_ASYNC_BOOST_ENTER_MA) ? 1U : 0U;
    }
```

## 6. 为什么这样改（不是别的改法）

- **为什么不直接把 Q2 永远同步**：CC 极轻载或软启动头几个周期，若强制同步整流，电感可能出现反向电流（电池 → 变换器倒灌）。保留一个很低的兜底阈值（40mA 进 / 80mA 退）让这些极轻载场景仍走异步，更稳。CV/其他模式仍沿用原来的通用异步整流逻辑。
- **为什么阈值取 40/80mA**：正常 CC 充电电流（0.12A 起步、目标 0.6A）都 > 80mA → 始终同步整流，问题解决；只有真正掉到 40mA 以下才进异步。进入/退出留 40mA 迟滞，避免边界抖动。
- **为什么不动升压裕量（800mV）或 duty 钳位**：那些是为防过冲/OVP 设的，治标且会带来过冲风险。根因是体二极管压降，直接让它走沟道（同步）才是对症。
- **为什么只改 CC 分支**：预连接、AFE handover、数字电源各有独立的异步整流策略（见同函数其他分支），它们的轻载需求不同，不应被这次改动影响。

## 9. 风险与回退

- **风险**：同步整流下若控制异常要求极小电流，理论上可能出现电感反向电流。已用 40mA 兜底阈值缓解；CC 环主动控流 + OCP + duty 钳位进一步兜底，正向充电工况风险很低。
- **回退**：把改动 2 的 `POWER_CC_ASYNC_BOOST_ENTER_MA/EXIT_MA` 改回 `POWER_ASYNC_BOOST_CURRENT_MAX_MA/EXIT_CURRENT_MA` 即恢复原行为。改动集中在一个函数 + 两个宏，回退无副作用。

## 10. 待确认（如修复后电流仍上不去）

若改完电流仍卡，说明可能仍有路径阻抗偏大、Q2 未实际导通或另有限流，按序查：
1. 量 Q2 的 Vgs，确认同步整流后 Q2 确实被完整驱动导通；
2. 量 BM2016 主 CHG/DSG 的 Vgs，确认主 FET 完全增强导通；
3. 复查 duty 是否被 [Power_Control_Limit_Battery_Boost_Duty](Driver/power_control.c#L1215) 或轻载 duty 上限钳住（看 Duty 是否顶在某个固定值不动）。

---

## 11. 倒灌后的下一步安全验证方向

低阈值同步整流方案出现倒灌后，下一轮不要再通过“提前打开 Q2 同步整流”来解决 0.12A 卡死。更安全的方向是**保持 Boost 异步整流**，利用 Q2 体二极管天然阻断 VOUT 向 VIN 的反向电流，然后排查为什么电流环没有把 Boost 低边 duty 拉高到足够值。

建议按下面顺序验证：

1. **确认已经刷回回退固件**：`Power Debug` 中低电流 CC/Boost 应仍显示异步，且直流稳压电源不再出现反向电流。
2. **看 input guard 是否在压 duty**：当前 `POWER_INPUT_GUARD_IIN_MAX_MA = 1800`。异步/轻载 DCM 下 `ADC_Iin` 可能虚高，如果 `Iin` 读数超过 1.8A，`Power_Control_Reduce_Duty_For_Input_Guard()` 会持续减 duty，表现就是电流环想加 duty，但 duty 被保护逻辑打下来。
3. **看 duty 是否顶在 Boost 上限**：当前 CC/涓流下 `Power_Control_Cc_Duty_Target_Mv()` 用 `max(pack+800mV, Vout+800mV)` 计算 duty 目标。如果 duty 顶住上限但电流仍只有 0.12A，下一步应考虑只提高 CC 的异步 Boost 裕量，例如把 `POWER_CC_DUTY_PACK_MARGIN_MV/POWER_CC_DUTY_VOUT_ADVANCE_MV` 做台架试验性提高，而不是打开 Q2 同步整流。
4. **确认 BM2016 电流方向**：CC 电流环用 BM2016 `batteryCurrentMa` 做反馈；若方向、滤波或路径切换异常，PI 可能得到错误误差。
5. **若必须做同步整流**：需要先增加反向电流判据或零电流检测策略，例如检测 `batteryCurrentMa <= 0`、`inputCurrentMa < 0/电源反灌`、或者 Boost 关断相位电流过零后立即退出同步。没有这些保护前，不建议在接真实电池时打开 Q2 同步整流。

下一版更合理的候选修复不是 40mA/80mA，而是：

```text
保持 Q2 异步关断
  -> 放宽/确认 input guard
  -> 增大 CC 异步 Boost duty 裕量
  -> 保留 OVP/OCP
  -> 小电流限流台架逐步验证
```

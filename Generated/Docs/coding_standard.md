# GD32G553 代码规范落地说明

本工程遵循 `../code.md` 中的嵌入式 Agent 代码规范。

## 执行规则

1. 先计划，再实现。
2. 先更新设计文档，再修改代码。
3. 按职责分层组织代码。
4. 复用现有 FreeRTOS、GD32 标准外设库和本地模块。
5. 修改保持最小、可回退。
6. 每次变更添加或更新测试。
7. 交付前运行可用测试。
8. 使用 Git 分支管理，并在提交前检查差异。

## 分层规则

| 层级 | 目录 | 职责 |
|---|---|---|
| App | `GD32G553/App` | FreeRTOS 任务创建和周期调度 |
| BSP | `GD32G553/BSP` | 板级 UART、LED、毫秒计时封装 |
| Common | `GD32G553/Common` | 共享类型、电池模型、通用 PI 控制器 |
| Config | `GD32G553/Config` | 板级映射、标定常量、保护默认值 |
| Driver | `GD32G553/Driver` | AFE、ADC、PWM、Flash 等硬件访问 |
| Module | `GD32G553/Module` | 充电状态机、SOC、均衡、电源路径策略 |
| Service | `GD32G553/Service` | 通信、状态快照、故障日志、参数存储、安全服务 |
| Tests | `Generated/Tests` | 主机侧检查和单元测试 |
| Docs | `Generated/Docs` | 架构、设计、变更和测试记录 |

## 当前边界

现有充电控制逻辑仍包含少数长文件。本次重构只调整目录和构建归属，不拆分已上板验证的硬件控制细节，避免把结构治理和行为变更混在一起。后续应按稳定边界逐个模块拆分，并保留硬件回归记录。
